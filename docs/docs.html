El lenguaje HTML

HTML es un lenguaje; no de programación sino de estructura de una página web. Nos permite situar elementos dentro de la página siguiendo la lógica y estructura del modelo conocido como DOM(Document Object Model). Los elementos HTML poseen "atributos" que permiten personalizarlos y/o manipularlos. Hay atributos comunes a todos los elementos HTML y existen otros que sólo aplican a cierto tipo de elementos. Es cuestión de ir familiarizándose con el lenguaje para poder acostumbrarse a ellos.

El lenguaje CSS

Es un lenguaje que permite modificar la apariencia de los elementos de una página web. Todo elemento HTML tiene ciertas "propiedades". CSS permite modificar los valores de las mismas que tienen por defecto, según necesitemos. Mediante éstos podemos modificar tipo de texto, fondo, colores, posición, y más.

Ahora bien. Para poder modificar un elemento es necesario indicar a qué elemento se va a aplicar la modificación. Justamente para esto existen los selectores CSS que no son más que distintas "formas" de referirnos a los elementos. Existen selectores que identifican elementos de forma individual (id) o por grupo, y dentro de la identificación por grupo podemos agrupar a todos los elementos de cierto tipo (nombre de elemento) o, incluso, podemos definir nosotros mismos los grupos (clases). Dependiendo de lo que necesitemos hacer, elegiremos el selector apropiado.

Selectores

  • ID

    Debe ser único en toda la página.

    En un elemento html agregamos un ID por medio del atributo HTML "id". En código CSS identificamos un ID con la sintaxis "#nombre_id"

  • Clase

    Asociamos un elemento html a una clase por medio del atributo HTML "class". Un elemento puede pertenecer simultáneamente a maś de una clase, en tal caso separamos los nombres con un espacio. En código CSS identificamos una clase con la sintaxis ".nombre_clase"

  • Nombre de elemento

    En código CSS identificamos a todos los elementos HTML con la sintaxis "nombre_elemento"

Usando selectores

  • A, B { ... }

    Cambios aplican para A y para B.

  • A B { ... }

    Cambios aplican para todo B que tenga A como padre.

  • A > B { ... }

    Cambios aplican para todo B que que sea HIJO DIRECTO de A.

  • A + B { ... }

    Cambios afectan sólo al primer B que esté luego de A, si es que hay uno.

  • A ~ B { ... }

    Cambios afectan a todo B que venga luego de un A.

  • [X=Y] { ... }

    Cambios afectan a todo atributo X que tenga valor Y.

  • X:Y { ... }

    Cambios afectan a las pseudocalses de los elementos html.

  • X::Y { ... }

    Cambios afectan a los pseudoelementos de los elementos html.

CSS Responsive

Cuando decimos que un sitio es responsivo nos referimos a que se adapta o responde distinto según características del dispositivo en el que se esté mostrando.

Una primera opción para acercarnos a lo responsive es utilzar media-queries. Esto nos va a permitir definir comportamiento CSS en base a las medidas de la pantalla del equipo que esté abriendo un sitio. Con la definición de bloques @media podemos hacer distintos bloques CSS según el dispositivo. Incluso podríamos definir distintos archivos ".css" según el equipo que visite el sitio. La sintaxis dentro de un bloque @media no varía del código css normal.

SCSS

Gracias a una poderosa gema Ruby llamada sass podemos utilizar código SCSS para definir archivos CSS de una forma más prolija y en más alto nivel. Los archivos .scss son compilados y transformados en .css para poder ser leídos por los navegadores. El código SCSS es similar al CSS pero maneja de forma más clara aspectos como la herencia de propiedades CSS, definiciones de variables, bloques @media, etc.

Para que la gema actúe es necesario correr el comando sass --watch nombreArch.scss:nombreArch.css

Ejemplo de herencia:

%padre{
      background-color: yellow;
}
.claseHijo {
      @extend %padre;
      color: blue;
}

Ejemplo de variable:

$color: red;
.clase {
      background-color: $color;
}

Ejemplo de nesting donde todos los elementos de la clase2 tendrán fondo azul SÓLO SI están dentro de elementos de la clase1:

.clase1 {
      .clase2{
            background-color: blue;
      }
}

Importación de módulos

Al momento de importar un módulo, Python literalmente copia y pega el código del archivo que se está trayendo y ejecuta una a una las línas que tenga escritas. Esto puede llevar a ocasionar problemas porque se corren sentencias que no esperamos que tengan lugar.

La forma correcta de controlar que esto suceda es definir una función en el archivo ".py" como la siguiente:

# ...código python
def main():
   # Comportamiento esperado para el archivo cuando sea éste quien se ejecute.
if __name__ == "__main__":
   main()

Así, Python ejecutará el código correspondiente a la función main() sólo si el archivo es el que fue ejecutado. Se conoce el nombre del archivo ejecutado por medio de la variable global __name__.

Flask

Flask es un microframework escrito en Python que facilita la creación y puesta en marcha de aplicaciones web. A su vez proporciona módulos que son útiles al momento de atender pedidos HTTP y transferir información en distintos formatos (JSON, Templates, etc.).

Por defecto el servidor Flask corre en el puerto 5000.

Por lo general una aplicación Flask contiene métodos que responden a los requests HTTP. La definición del formato de cada request se hace por medio de decoradores. A continuación, un código que responde a pedidos HTTP GET y POST con un mensaje "Hello World!".

@app.route('/', methods=['GET', 'POST'])
def index():
    return "Hello World!"

Módulos y objetos Flask

  • render_template (objeto)

    Permite devolver una página .html como response. El template se buscará dentro de la carpeta templates ubicada en la raíz del proyecto, es decir, donde esté el archivo "*.py" que contenga la aplicación Flask. La sentencia render_template admite el pasaje de parámetros desde el servidor hacia la vista.

    @app.route('/algunaRuta', methods=['GET', 'POST'])
    def nombreMetodo():
        mensaje = 'Hello World!'
        return render_template('vista.html', nombre_parametro=mensaje)

  • request (objeto)

    Es usado para la comunicación entre un template y el servidor cuando se hace una petición a una url por POST, lo que significa que el servidor recibe datos que NO FIGURAN en la url de la petición. El objeto request contiene un objeto form el cual permite obtener el valor de un atributo name de un input por medio del método get.

    valor = request.form.get('nombreDelCampoEnAtributoName')

  • flask_session (módulo)

    Configura el servidor para habilitar el uso de variables de sesión. Contiene el objeto Session el cual requiere una configuración básica y una asociación con la aplicación Flask.

    app.config["SESSION_PERMANENT"] = False
    app.config["SESSION_TYPE"] = "filesystem"
    Session(app)

  • session (objeto)

    Trabaja junto con el módulo flask_session. Permite acceder al almacenamiento de datos en la sesión. Una vez configurado el uso de sesión, la variable session se accede como un diccionario Python común.

    # inicialización de variable de sesión llamada "nombres"
    if session.get("nombres") is None:
       session["nombres"] = []

    Variable booleana que al valer True permite que Flask recargue la aplicación cada vez que se detecta un cambio en el código del servidor.

Formularios

Así como el servidor envía datos a un template, éste puede enviarle datos al server. Una de las alternativas para lograr la comunicación en este sentido es el uso de formularios. Los formularios son elementos HTML que contienen elementos de tipo "entrada" y son útiles para obtener datos de un usuario del sistema.

Los elementos de entrada son los input y con el atributo type logramos que se comporten del modo que queremos. Un detalle importante en la comunicación template -> server es que el servidor recibirá de un formulario el valor que esté especificado en el atributo name del elemento input. Cuando trabajamos con Flask, el modo correcto de recuperar datos desde un template en el servidor es por medio del módulo request.

Manejo de sesiones

Las sesiones permiten a un servidor almacenar datos asociados a un usuario particular. Recurrimos a ellas cuando hay cierta información que pertenece a un usuario y no queremos que los datos sean accesibles por otros.

Las variables de sesión mueren cuando el server se reinicia. Si lo que necesitamos es mayor persistencia hay que utilizar una base de datos.

Jinja2

Es el componente Flask que permite trabajar con vistas. Al momento de renderizar una vista Jinja2 se encarga de buscarlas y crearlas, reemplazando valores de parámetros enviados a las vistas desde el servidor y ejecutando sentencias de control siempre que corresponda, para lo cual cuenta con una sintaxis propia.

Acceso a variables

Debemos saber con qué nombre hemos pasado por medio de un "render_template" una variable a una vista. Suponiendo que la variable se llama "param1", mostramos cómo acceder al valor de la misma según corresponda.

Para usar el valor de la variable en una estructura de control

{% param1 %}

Para mostrar el valor de una variable en pantalla

{{ param1 }}

Herencia de vistas

La herencia entre templates es muy útil cuando nuestra vista cuenta con zonas bien diferenciables o modularizables. La herencia nos permite definir bloques del código de una vista que deben ser completados con el contenido especificado en un archvio .html diferente al actual. Esto nos brinda reusabilidad de código y hace más legible y más prolijo el proyecto.

Cuando usamos herencia de vistas necesitamos incluir código Jinja2 en dos archivos .html distintos: uno que marca dónde se introducirá el bloque "comodín" y el otro que define el contenido .html que debe ser insertado al momento de renderizar un template.

En el template principal...

"...codigo html..."
{% block nombreBloque %} {% endblock %}
"...codigo html..."

En el template externo...

{% extends 'nombreArchivo.html' %}
{% block nombreBloque %}
"...codigo html..."
{% endblock %}

SQLAlchemy

Es biblioteca Python que permite ejecutar comandos SQL desde Python.

Podemos usar SQLAlchemy para realizar consultas a una base de datos escribiéndolas en texto plano. Primero se importa el módulo, se crea el objeto "db", que será la conexión con la bdd.

from sqlalchemy import create_engine
from sqlalchemy.orm import scoped_session, sessionmaker
engine = create_engine('conection_url')
db = scoped_session(sessionmaker(bind=engine))

Base de datos

Hay dos tipos de bases de datos: relacionales y no relacionales. La distinción surge de la forma en la que se accede a los datos. En las relacionales se cuenta con una referencia al atributo identificador de un elemento de una tabla externa.

SQL es el lenguaje principal de las base de datos relacionales. Sus siglas vienen de Structured Queries Language. Ejemplos de bases de datos SQL son SQLite, MySQL, PostgreSQL.

Un detalle importante es que SQL se considera un estándar que, según el motor de base de datos que se utilice, se implementará de una u otra manera. A grandes rasgos, la esencia SQL se respeta, aunque con ciertas variaciones.

Tipos de datos

  • VARCHAR(xx) -> strings de xx largo
  • TIMESTAMP -> fecha y hora
  • BOOLEAN
  • ENUM
  • ...

Constraints

Son restricciones que se imponen a las columnas de una tabla de la base de datos.

  • NOT NULL -> Campo no puede ser vacío.
  • UNIQUE -> Valor único de columna para toda la tabla.
  • PRIMARY KEY -> Campo identificador único de cada registro de la tabla.
  • DEFAULT -> Valor por defecto para la columna, en caso que no se especifique algún valor.
  • CHECK -> Usados para cumplir cierta condición.
  • REFERENCES -> Usado con claves foráneas. Referenciamos otra tabla.
  • ...

Race conditions

Si mucha gente quiere acceder a la BDD y hacer operaciones a la vez podríamos tener problemas. El problema ocurre cuando se nos meten operaciones en el medio de una más grande. Para solucionarlo existen las transacciones. Éstas permiten bloquear el acceso de la BDD para que otras operaciones no molesten.

BEGIN -- bloquea la bdd
COMMIT -- libera la bdd
