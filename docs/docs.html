Archivos .CSV

Los archivos CSV son utilizados para manejar datos. Cada línea es un dato y cada atributo está separado por una coma. De ahí el nombre: Comma Separated Values.

Para trabajar este tipo de archivos desde Python se puede importar el módulo llamado csv.

HTTP

Las siglas HTTP refieren a HyperText Transfer Protocol. Gracias a él funciona internet. Es el protocolo utilizado para comunicar servidores.

Métodos HTTP y sus significados

  • GET: tomar un recurso
  • POST: crear un nuevo recurso
  • PUT: modificar un recurso
  • PATCH: actualizar un recurso
  • DELETE: borrar un recurso

Making Requests

Desde Python, con el modulo "requests" podemos hacer distintos requests con cada tipo de metodo http. request.text() devuelve el contenido de texto plano de un sitio.

Hacer un request de tipo GET con parámetros (opcionales) a una url.

requests.get('url', params={'param1': valor1, 'param2': valor2})

Hacer un request de tipo POST a una url.

requests.post('url')

Status codes

Cuando hacemos un pedido http la response contiene un código. Algunos de ellos son:

  • 200 OK
  • 201 Created
  • 400 Bad request
  • 403 Forbidden
  • 404 Not found
  • 405 Method not allowed
  • 422 Unprocessable entity

API

Podemos entender a las APIs como protocolos que mantienen las aplicaciones para interactuar entre sí para enviar o recibir informacion. Es la interfaz que una aplicacion tiene para funcionar con otras. Una API entiende qué es lo que el usuario quiere por medio de la combinación URL + Método HTTP. Las aplicaciones trabajan enviando y recibiendo recursos.

API KEY

Muchas veces tenemos restringido el acceso a una api por medio de una API Key. Esto se hace para no saturar el acceso. Además, gracias a la key podemos controlar qué información retornarle a un usuario dado, al acceder a la api.

El lenguaje HTML

HTML es un lenguaje; no de programación sino de estructura de una página web. Nos permite situar elementos dentro de la página siguiendo la lógica y estructura del modelo conocido como DOM(Document Object Model). Los elementos HTML poseen "atributos" que permiten personalizarlos y/o manipularlos. Hay atributos comunes a todos los elementos HTML y existen otros que sólo aplican a cierto tipo de elementos. Es cuestión de ir familiarizándose con el lenguaje para poder acostumbrarse a ellos.

La estructura de una página web responde al modelo conocido como DOM (Document Object Model). Existe un elemento raíz, del cual nacen los demás. Los diferentes tags representan elementos y cada uno puede contener otros y así siguiendo.

El lenguaje CSS

Es un lenguaje que permite modificar la apariencia de los elementos de una página web. Todo elemento HTML tiene ciertas "propiedades". CSS permite modificar los valores de las mismas que tienen por defecto, según necesitemos. Mediante éstos podemos modificar tipo de texto, fondo, colores, posición, y más.

Ahora bien. Para poder modificar un elemento es necesario indicar a qué elemento se va a aplicar la modificación. Justamente para esto existen los selectores CSS que no son más que distintas "formas" de referirnos a los elementos. Existen selectores que identifican elementos de forma individual (id) o por grupo, y dentro de la identificación por grupo podemos agrupar a todos los elementos de cierto tipo (nombre de elemento) o, incluso, podemos definir nosotros mismos los grupos (clases). Dependiendo de lo que necesitemos hacer, elegiremos el selector apropiado.

Selectores

  • ID

    Debe ser único en toda la página.

    En un elemento html agregamos un ID por medio del atributo HTML "id". En código CSS identificamos un ID con la sintaxis "#nombre_id"

  • Clase

    Asociamos un elemento html a una clase por medio del atributo HTML "class". Un elemento puede pertenecer simultáneamente a maś de una clase, en tal caso separamos los nombres con un espacio. En código CSS identificamos una clase con la sintaxis ".nombre_clase"

  • Nombre de elemento

    En código CSS identificamos a todos los elementos HTML con la sintaxis "nombre_elemento"

Usando selectores

  • A, B { ... }

    Cambios aplican para A y para B.

  • A B { ... }

    Cambios aplican para todo B que tenga A como padre.

  • A > B { ... }

    Cambios aplican para todo B que que sea HIJO DIRECTO de A.

  • A + B { ... }

    Cambios afectan sólo al primer B que esté luego de A, si es que hay uno.

  • A ~ B { ... }

    Cambios afectan a todo B que venga luego de un A.

  • [X=Y] { ... }

    Cambios afectan a todo atributo X que tenga valor Y.

  • X:Y { ... }

    Cambios afectan a las pseudocalses de los elementos html.

  • X::Y { ... }

    Cambios afectan a los pseudoelementos de los elementos html.

CSS Responsive

Cuando decimos que un sitio es responsivo nos referimos a que se adapta o responde distinto según características del dispositivo en el que se esté mostrando.

Una primera opción para acercarnos a lo responsive es utilzar media-queries. Esto nos va a permitir definir comportamiento CSS en base a las medidas de la pantalla del equipo que esté abriendo un sitio. Con la definición de bloques @media podemos hacer distintos bloques CSS según el dispositivo. Incluso podríamos definir distintos archivos ".css" según el equipo que visite el sitio. La sintaxis dentro de un bloque @media no varía del código css normal.

SCSS

Gracias a una poderosa gema Ruby llamada sass podemos utilizar código SCSS para definir archivos CSS de una forma más prolija y en más alto nivel. Los archivos .scss son compilados y transformados en .css para poder ser leídos por los navegadores. El código SCSS es similar al CSS pero maneja de forma más clara aspectos como la herencia de propiedades CSS, definiciones de variables, bloques @media, etc.

Para que la gema actúe es necesario correr el comando sass --watch nombreArch.scss:nombreArch.css

Ejemplo de herencia:

%padre{
      background-color: yellow;
}
.claseHijo {
      @extend %padre;
      color: blue;
}

Ejemplo de variable:

$color: red;
.clase {
      background-color: $color;
}

Ejemplo de nesting donde todos los elementos de la clase2 tendrán fondo azul SÓLO SI están dentro de elementos de la clase1:

.clase1 {
      .clase2{
            background-color: blue;
      }
}

Animaciones

Podemos dar la ilusión de tener elementos html animados usando CSS. Una animación no es más que instrucciones CSS que permiten establecer para un cierto elemento, propiedades de un estado inicial y de uno final, y se indica en cuánto tiempo se necesita que se llegue de uno a otro.

En un archivo .css definimos una animacion css con @keyframes y dentro pondremos dos estados: inicial (from) y final (to). Dentro de cada estado indicamos los valores de los atributos que deberá tener un elemento animado.

Si nuestra animación requiere más de dos estados, podemos establecer el comportamiento CSS especificando porcentaje de tiempo de duración.

Ej: agrandamos un titulo de 20px a 100px en un lapso de 2 segundos.

@keyframes my_animation_name {
   from {
      ... propiedades a definir para estado inicial
      font-size: 20px:
   }
   to {
      ... propiedades a definir para estado final
      font-size: 100px:
   }
}
h1 {
   animation_name: my_animation_name;
   animation-duration: 2s;
   animatino-fill-mode: forwards;
}

Más propiedades de una animación CSS
  • animation-iteration-count: infinite;

Python

Esta sección contiene algunos datos útiles sobre el lenguaje. Python es usado en esta ocasión como lenguaje en el lado del servidor. No se espera que esta sección enseñe Python sino que se quiere resaltar aspectos que en lo personal considero importantes.

Valor por defecto en variable

Python permite establecer un valor por defecto a una variable, funcionalidad por de más útil cuando esperamos un dato del client-side que puede venir vacío.

start = int(request.form.get('start') or 0)

En este caso, si no existe un elemento del form con name='start' con contenido, la variable start valdrá 0.

Imprimir en pantalla con formato

Para imprimir variables junto con texto a partir de Python3 podemos usar:

nombre = 'Lucas'

En general comparten las características de almacenar datos de cualquier tipo, a la vez, y se acceden por medio de un índice. Al momento del acceso de información todas las estructuras de Python, salvo los sets, lo hacen por medio de un índice entre corchetes "[]".

Clases

Python permite trabajar con objetos mediante la definición de clases. En general una clase se define así:

class nombreClase(clasePadreSiLaHubiere):

Todo objeto python debe incluir como primer parámetro una referencia a sí mismo. Esta es justamente la función del parámetro self, el cual no debe llamarse así por alguna razón, pero se considera una convensión.

Importación de módulos

Al momento de importar un módulo, Python literalmente copia y pega el código del archivo que se está trayendo y ejecuta una a una las línas que tenga escritas. Esto puede llevar a ocasionar problemas porque se corren sentencias que no esperamos que tengan lugar.

La forma correcta de controlar que esto suceda es definir una función en el archivo ".py" como la siguiente:

# ...código python
def main():
   # Comportamiento esperado para el archivo cuando sea éste quien se ejecute.
if __name__ == "__main__":
   main()

Así, Python ejecutará el código correspondiente a la función main() sólo si el archivo es el que fue ejecutado. Se conoce el nombre del archivo ejecutado por medio de la variable global __name__.

  • NameError

Flask

Flask es un microframework escrito en Python que facilita la creación y puesta en marcha de aplicaciones web. A su vez proporciona módulos que son útiles al momento de atender pedidos HTTP y transferir información en distintos formatos (JSON, Templates, etc.).

Por defecto el servidor Flask corre en el puerto 5000.

Por lo general una aplicación Flask contiene métodos que responden a los requests HTTP. La definición del formato de cada request se hace por medio de decoradores. A continuación, un código que responde a pedidos HTTP GET y POST con un mensaje "Hello World!".

@app.route('/', methods=['GET', 'POST'])
def index():
    return "Hello World!"

Formularios

Así como el servidor envía datos a un template, éste puede enviarle datos al server. Una de las alternativas para lograr la comunicación en este sentido es el uso de formularios. Los formularios son elementos HTML que contienen elementos de tipo "entrada" y son útiles para obtener datos de un usuario del sistema.

Los elementos de entrada son los input y con el atributo type logramos que se comporten del modo que queremos. Un detalle importante en la comunicación template -> server es que el servidor recibirá de un formulario el valor que esté especificado en el atributo name del elemento input. Cuando trabajamos con Flask, el modo correcto de recuperar datos desde un template en el servidor es por medio del módulo request.

Manejo de sesiones

Las sesiones permiten a un servidor almacenar datos asociados a un usuario particular. Recurrimos a ellas cuando hay cierta información que pertenece a un usuario y no queremos que los datos sean accesibles por otros.

Las variables de sesión mueren cuando el server se reinicia. Si lo que necesitamos es mayor persistencia hay que utilizar una base de datos.

SQLAlchemy

Es biblioteca Python que permite ejecutar comandos SQL desde Python.

Podemos usar SQLAlchemy para realizar consultas a una base de datos escribiéndolas en texto plano. Primero se importa el módulo, se crea el objeto "db", que será la conexión con la bdd.

from sqlalchemy import create_engine
from sqlalchemy.orm import scoped_session, sessionmaker
engine = create_engine('conection_url')
db = scoped_session(sessionmaker(bind=engine))

Flask-SQLAlchemy

Un ORM (Object-Relational Mapping) permite realizar operaciones contra una base de datos desde el código de un servidor, de una forma "alto nivel". Flask-SQLAlchemy es un módulo creado para comunicar una aplicacion Flask con SQLAlchemy, el ORM de Python. Facilita la configuración del ORM y estrucuta la vinculación de los modelos de nuestro sistema.

Podemos usar Flask-SQLAlchemy agregamos el módulo correspondiente, creamos la aplicación Flask, asociamos un url de conexión con la bdd a la aplicación Flask y, por último, creamos un objeto base de datos al que le asociamos la aplicación en el constructor.

from flask_sqlalchemy import SQLAlchemy app = Flask(__name__) app.config['SQLALCHEMY_DATABASE_URI'] = 'postgresql+psycopg2://userName:userPassword@host:port/dbName' db = SQLAlchemy(app)

Flask-SocketIO

Permite establecer canales de comunicación ágil entre un servidor (python) y un cliente (javascript). Importamos el objeto SocketIO y el objeto emit así:

from flask_socketio import SocketIO, emit

Flask sabrá con qué método responder a cada evento de un socket gracias al uso de un decorador, el cual recibe como argumento el nombre del evento a escuchar. El método puede recibir datos como parámetros en formato JSON (o diccionario). En caso de tener que responder con datos al client-side, el objeto emit podrá generar un evento y enviar datos al mismo tiempo. Mediante el parámetro broadcast podemos especificar si la el mensaje deberá afectar a todos los escuchas del evento o no.

# El socket recibe de parte del cliente un evento de nombre 'submit vote'
@socketio.on('submit_vote')
def vote(data):
   selection = data["selection"]
   votes[selection] += 1
   # Emitimos un evento, una señal, de nombre 'announce vote'
   # el parametro broadcast=True permite enviar la informacion a todos los sockets que esten escuchando, incluyendo el usuario actual
   emit('announce vote', {'yes': votes['yes'], 'no': votes['no'], 'maybe': votes['maybe']}, broadcast=True)

Base de datos

Hay dos tipos de bases de datos: relacionales y no relacionales. La distinción surge de la forma en la que se accede a los datos. En las relacionales se cuenta con una referencia al atributo identificador de un elemento de una tabla externa.

SQL es el lenguaje principal de las base de datos relacionales. Sus siglas vienen de Structured Queries Language. Ejemplos de bases de datos SQL son SQLite, MySQL, PostgreSQL.

Un detalle importante es que SQL se considera un estándar que, según el motor de base de datos que se utilice, se implementará de una u otra manera. A grandes rasgos, la esencia SQL se respeta, aunque con ciertas variaciones.

Tipos de datos

  • INTEGER
  • DECIMAL
  • SERIAL -> integer autoincrementable / bastante particular de postgreSQL
  • VARCHAR(xx) -> strings de xx largo
  • TIMESTAMP -> fecha y hora
  • BOOLEAN
  • ENUM
  • ...

Constraints

Son restricciones que se imponen a las columnas de una tabla de la base de datos.

  • NOT NULL -> Campo no puede ser vacío.
  • UNIQUE -> Valor único de columna para toda la tabla.
  • PRIMARY KEY -> Campo identificador único de cada registro de la tabla.

  • CHECK -> Usados para cumplir cierta condición.
  • REFERENCES -> Usado con claves foráneas. Referenciamos otra tabla.

JavaScript

El funcionamiento de internet se basa en el paradigma cliente-servidor. Generalmente, el servidor es muy potente en cuestiones de performance pero aún así puede saturarse si es muy demandado. JavaScript (JS) es uno de los lenguajes más utilizados del lado del cliente y la razón de su existencia se relaciona con disminuir la demanda (la carga) de un servidor, lograr la ejecución de operaciones rápidas evitando los tiempos de comunicación agregados por la red.

Cada navegador "es un mundo" en cuanto al entendimiento de JavaScript por lo que se recurre a la adopción de estándares, que no son más que reglas para que todos nos pongamos de acuerdo sobre versiones de JS que todo navegador debe entender por igual. Hoy en día el estandar más popular es el "ES6".

El código JS se incluye en un archivo html. Puede escribirse directamente en él o importarse, al igual que el código CSS. Las sentencias JS se incluyen dentro del tag script y cada una termina con punto y coma ";".

JS permite manipular objetos del DOM para modificar su comportamiento o la estructura general de la página de forma programática, es decir, dinámica.

En JS podemos referirnos a la página actual como un documento. De hecho, eso es lo que es: un documento ".html". Lo hacemos por medio de una variable global llamada document.

Variables JS

Las formas de definir variables según alcance son tres: const, let y var.

  • const -> no puede cambiar su valor. Alcance limitado al bloque donde se declara.
  • let -> variable que no existe fuera del bloque donde fue declarada.
  • var -> vive en toda la funcion donde fue declarada.

Comparación JS

El símbolo "===" es el que usaremos en JS para comparar variables ya que éste no sólo compara el contenido de dos variables sino también su tipo de dato

Local Storage

JS nos permite almacenar información en la pc del usuario mediante LOCAL STORAGE. El elemento localStorage permite guardar información con el formato

localStorage.setItem('key', value);

recuperarla con

localStorage.getItem('key');

y borrarla con

localStorage.removeItem('key');

Las keys del localStorage se almacenan según el nombre de domino, evitando pisar valores de igual clave para distintos sitios web. Sin embargo, hay que tener cuidado si usamos localStorage con igual nombre de claves en dos páginas distintas pero en el mismo dominio, como por ejemplo la casa de uno, para no pisar valores.

AJAX - Client

AJAX (Asincronic Javascript And XML) nos permite generar requests para pedir información a un servidor sin tener que recargar la página completa. Una de las formas de usar AJAX es enviar información al servidor por medio de un formulario y recibir una response JSON.

Creación de un objeto request para realizar un pedido asincrónico a un servidor.

const request = new XMLHttpRequest();
request.open('POST', '/ajax/chats/');

Preparación y envío de datos al servidor en formato strong>'key': value.

const data = new FormData();
data.append('key1', value1);
data.append('key2', value2);
request.send(data);

Definir código a ejecutar cuando el request haya sido atendido por el server-side.

request.onload = () => {
   const data = JSON.parse(request.responseText);
   console.log(data.key1);
   console.log(data.key2);
};

Web sockets

Cuando hablamos de comunicación cliente-servidor, el servidor responde a cada pedido del cliente. Si no hay pedidos no hay respuestas. En una aplicacion de chat, por ejemplo, nosotros quisieramos no tener que actualizar la página para ver nuevos mensajes. En realidad nos gustaría que la actualización fuese en tiempo real (full-duplex comunication).

Socket.IO es biblioteca JS que permite hacer esto. La importamos agregando esta línea dentro del head del documento HTML.

<\script src="//cdnjs.cloudflare.com/ajax/libs/socket.io/2.2.0/socket.io.js" integrity="sha256-yr4fRk/GU1ehYJPAs8P4JlTgu0Hdsp4ZKrx8bDEDC3I=" crossorigin="anonymous"><\/script>

Creamos un objeto socket

// conectar el socker entre el cliente y el server
// La siguiente linea es standar
var socket = io.connect(location.protocol + '//' + document.domain + ':' + location.port);

Un objeto socket responde a eventos. Algunos son definidos por la biblioteca y otros por el programador con un simple String. Definimos un comportamiento para el socket para cuando se haya conectado con el server-side (lo logramos con el evento "connect"). En el caso que sigue cuando el socket se haya conectado, definimos que todos los botones de la página al ser clickeados deberán crear una variable llamada "selection" la cual almacenará el voto que represente el atirbuto "data" del botón que haya sido clickeado. Luego, se indica al socket en el client-side que genere un evento, definido por el programador, llamado "submit_vote" y que conlleva un parámetro que es simplemente un diccionario con el valor de la variable "selection".

// El objeto socket responde a eventos
// Cuando el socket se conecte...
socket.on('connect', () => {
   document.querySelectorAll('button').forEach(button => {
      button.onclick = () => {
         const selection = button.dataset.vote;
         // El objeto socket emite un evento de nombre 'submit vote'
         // Lo que se envía al server es el diccionario que se esta pasando como segundo parámetro
         socket.emit('submit_vote', {'selection': selection})
      };
   });
});

Single vs Multi pages

Esta sección muestra dos arquitecturas de páginas web: single page y multi page. La primera consiste en mantener secciones de una página como contenido dinámico, y en base al accionar del usuario o al paso del tiempo, ir modificándolo. La segunda, en cambio, trata de muchas páginas diferentes donde, si queremos cambiar el contenido, necesariamente debemos cambiar de página. Estos conceptos en la práctica pueden convivir.

La ganancia de una arquitectura single-page está en que podemos modificar SÓLO una porción de toda la página ganando tiempo de carga y respuesta en el sitio. También se nos suma complejidad de código frente a la opción multi-page. Un ejemplo de esto es que en single-page deberemos simular "manualmente" el cambio de url ya que el navegador entenderá que siempre permanecemos en el mismo sitio.

HTML5 History API

¿Cómo podemos simular ese cambio de url sin que la página deba recargarse completamente? En HTML5 tenemos lo que se llama "HTML5 History API". Es una funcionalidad de HTML que nos permite conocer el historial del navegador y actualizar la URL para mostrar lo que quiero. La forma de conseguirlo es hacer un PUSH del estado de la URL al historial del navegador. Esto se controla con JavaScript.

// history.pushState(param1, param2, param3);
   // param1: dato que quiero asociar con el push
   // param2: titulo de la pag que queremos pushear
   // param3: URL que queremos pushear

EJ:

history.pushState(null, name, name);

Esta es la forma correcta de cambiar la URL en single-page.

Si dejamos sólo estas líneas tendríamos un problema: si presiono la flecha 'atras' del navegador, la URL cambiará pero no el contenido pertinente para esa URL. Esto se debe a que no tenemos un codigo JS que le indique al navegador qué hacer si se presiona la flecha 'atras'. Para arreglarlo necesitamos hacer un 'pop' a la pila de urls de la history API. La API dispone de un evento que se llama 'onpopstate' y es justamente lo que necesitamos para desapilar urls y darles un comportamiento.

Trabajamos con el objeto window y el método onpopstate.

window.onpopstate = e => {
   const data = e.state;
   document.title = data.title;
   document.querySelector('main').innerHTML = data.text;
};
